

Build a production-style web portal called Global Self-Pay Price Intelligence.

Core mission:
Create a search engine that finds actual posted out-of-pocket healthcare prices for specific services across the United States and internationally. This portal must prioritize real posted prices found on the public web or in public machine-readable pricing files. Do not return estimates, generic ranges, “typical cost” content, insurance-derived approximations, or AI-invented pricing. If an exact posted price is not found, say so clearly and do not fabricate one.

Design requirement:
The entire portal must use a premium Apple macOS Tahoe glass aesthetic. The UI should feel luminous, ultra-polished, layered, translucent, and high-end, with frosted glass panels, glowing depth, soft reflections, subtle blur, elegant transparency, and rich lighting. It should feel cinematic and intelligent, like a next-generation search console built inside a luminous glass operating system. Prioritize a bright, glassy, extra-luminous interface rather than a flat or generic dashboard look.

Product vision:
Create a cinematic intelligence database for healthcare self-pay pricing. Users should be able to search for any healthcare service and see matching clinics, hospitals, labs, imaging centers, urgent cares, dental offices, pharmacies, specialist practices, and other providers that publicly post actual cash-pay or self-pay pricing.

Primary requirements:
1. Search globally and in the United States.
2. Show results on an interactive world map and also in a dense intelligence-style results panel.
3. Every result must include source-backed evidence of the posted price.
4. The system must distinguish between:
   - exact posted self-pay price
   - exact posted discounted cash price
   - bundled package price
   - posted fee schedule item
   - non-qualifying source
5. Only qualifying price evidence should be surfaced as a valid result.
6. The app must support searching by:
   - service name
   - CPT/HCPCS code when available
   - specialty
   - provider/facility name
   - city/state/country
   - radius
   - provider category
7. Include strong filtering for:
   - country
   - U.S. state
   - city
   - distance radius
   - provider type
   - service type
   - cash-pay only
   - hospital only
   - clinic only
   - imaging only
   - lab only
   - urgent care only
   - dental only
   - telehealth if posted
8. Support a detail view for each provider showing:
   - provider/facility name
   - location
   - map
   - exact price found
   - currency
   - billing code if found
   - price context
   - source URL
   - source snippet
   - timestamp found
   - confidence classification based on evidence quality, not guesswork

Critical sourcing rules:
Prioritize exact public price sources in this order:
A. U.S. hospital price transparency machine-readable files, especially discounted cash price fields
B. Public provider websites with explicit self-pay/cash pricing
C. Public PDFs, chargemasters, fee schedules, price lists, menus, and brochures
D. Structured metadata like JSON-LD if it contains actual price data
E. Public healthcare directories and provider registries for enrichment only, not for invented pricing
F. Common Crawl only as a discovery/indexing layer
G. Search APIs and sitemap/robots discovery as crawl assistance

Do not treat the following as final prices:
- blog articles quoting ranges
- generic cost guides
- insurance negotiated estimates unless clearly labeled as self-pay cash price
- pages that only say call for pricing
- AI-generated guesses
- SEO junk pages with no direct provider evidence

Use and integrate these data sources intelligently:

United States:
- NPPES NPI Registry for provider and organization lookup
- CMS Data datasets
- CMS Care Compare for facility enrichment
- CMS hospital price transparency machine-readable files and compliance resources
- robots.txt and sitemap.xml discovery
- JSON-LD extraction on provider pages
- Common Crawl CDX API for discovery only
- PDF/table extraction libraries for public price sheets and MRF-linked files

International:
- public provider websites
- public clinic and hospital fee schedules
- public diagnostics and lab menus
- public radiology and imaging pricing pages
- public dental fee pages
- public pharmacy vaccination and travel clinic pricing pages
- structured web extraction from provider websites, PDFs, tables, and downloadable public documents
- multilingual search and extraction support where possible

Search engine behavior:
When a user searches for a service, the system should:
1. Normalize the query into likely variants, synonyms, plain-English forms, billing code candidates, and international wording variants.
2. Discover providers and facilities likely to offer the service.
3. Search only for publicly posted exact prices.
4. Score candidate evidence based on source quality.
5. Return only results with direct evidence.
6. Show non-price matches in a separate “found provider but no posted price” section.

Architecture requirements:
Build this as a modular portal with connector-based ingestion. Each source type should have its own connector so the system can expand over time.

Required modules:
- query normalization engine
- provider discovery engine
- public web search and discovery layer
- source fetcher
- price evidence extractor
- structured parser for JSON, HTML, PDF, and table data
- deduplication engine
- result ranking engine
- map visualization layer
- export layer
- admin diagnostics panel

Result ranking logic:
Higher priority:
- exact cash or self-pay price on provider website
- discounted cash price in hospital MRF
- exact fee schedule line item
- exact CPT-linked or service-linked price

Lower priority:
- vague service mention with no price
- generic article with ranges
- facility known to offer service but no actual posted price

Technical requirements:
- build on Replit
- use a clean full-stack structure
- make the backend modular and connector-driven
- use server-side scraping and parsing pipelines
- cache aggressively
- respect robots.txt where applicable
- implement rate limiting and retry logic
- build for maintainability and future expansion
- avoid hardcoded one-off scrapers whenever possible
- create a clear admin panel showing:
  - crawl health
  - source hit rate
  - extraction success rate
  - rejected results
  - stale results
  - broken sources

For the first working version, prioritize:
1. U.S. hospitals with actual discounted cash prices from machine-readable files
2. U.S. provider websites with explicit self-pay pricing
3. common imaging, lab, urgent care, and dental services
4. international provider websites that publish actual fee pages
5. strong evidence extraction and source citation per result

Important guardrails:
- never show estimated prices as exact prices
- never invent or infer a price that is not explicitly posted
- never summarize ranges as final result prices
- clearly separate exact prices from non-price provider matches
- optimize for factuality over volume
- optimize for real posted cash-pay intelligence, not generic healthcare SEO
